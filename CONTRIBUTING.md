# Contributing

This repository is a **license specification**, not a conventional code project.

There is no application binary, service runtime, or feature backlog in the usual
software sense. Most contributions here are about legal drafting quality,
interpretive clarity, internal consistency, examples, and supporting
documentation.

## Contributions we welcome

We welcome issues and pull requests for:

- drafting clarity
- ambiguous or confusing wording
- clause-to-clause consistency problems
- compatibility analysis
- enforceability concerns
- legal or interpretive discussion
- example improvements
- documentation improvements
- formatting and editorial cleanup

## Good issue reports

If you open an issue, please try to include:

- the exact clause, annex, or sentence you are discussing
- the scenario that exposes the ambiguity or problem
- why the current wording is unclear, risky, or hard to apply
- a proposed interpretation or revision, if you have one

Concrete examples are much more useful than broad statements that the license
"feels wrong" or "seems too strict."

## Pull request guidance

If you open a pull request:

- keep the scope narrow when possible
- explain the drafting problem you are solving
- cite the affected clause numbers
- explain any compatibility, enforceability, or policy trade-off
- update supporting documentation or examples when needed

For documentation-only changes, clarity wins. For license-text changes,
precision wins.

## Scope note

Issues for legal or interpretive discussion are welcome.

That said, discussion in this repository is not legal advice and does not create
an attorney-client relationship. If you intend to rely on MCL for real-world
commercial use, you should have the text reviewed by qualified counsel.
