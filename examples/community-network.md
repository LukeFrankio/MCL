# Community-network example

This example mirrors a community-oriented public project that wants distribution
and network reciprocity, anti-shim protection, and permissive internal use.

## Good fit

- Public infrastructure projects with hosted deployment options
- Projects that want reciprocal SaaS publication without cloud-binary release

## Bad fit

- Projects that want private-fork publication deadlines
- Projects that want a non-competing commercial restriction
