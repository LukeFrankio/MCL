# Strong-cloud example

This example shows a project that wants strong public reciprocity for hosted use,
cloud deployment artifacts, anti-shim rules, private-fork deadlines, and CLA
infrastructure.

## Good fit

- Hosted infrastructure or control-plane software
- Projects worried about shell publication and private fork cycling

## Bad fit

- Projects that want a simpler community posture
- Projects that do not want CLA administration
